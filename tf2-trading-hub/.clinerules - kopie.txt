# TF2-HA repository update workflow

When working on the TF2 Home Assistant add-on:

- Do not read the full dist/server.js unless explicitly required.
- Use targeted searches such as Select-String first.
- Prefer minimal targeted edits.
- After every patch, always run:
  - node --check .\tf2-trading-hub\dist\server.js
  - node --check .\tf2-trading-hub\dist\index.js
- Always verify version markers in:
  - config.yaml
  - package.json
  - run.sh
  - dist/server.js
  - dist/index.js
  - README/update notes if touched
- Never push broken code.
- Never push if node --check fails.
- Never push secrets, tokens, API keys or real credentials.
- Never change credential vault logic unless the user explicitly asks.
- Never change trading safety defaults unless the user explicitly asks.

Git workflow:
- After successful patch and checks, run:
  - git status
  - git diff --stat
- If the user explicitly says "commit", create a commit with a clear patch message.
- If the user explicitly says "push to GitHub", run git push.
- If the user does not explicitly say push, stop after commit and report what changed.
